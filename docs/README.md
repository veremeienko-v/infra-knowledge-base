# 📘 Global Documentation

## 📌 Опис
Цей розділ містить загальну документацію всієї інфраструктури, опис архітектури, стандартів та принципів роботи репозиторію.

Він є центральною точкою знань і використовується для:
- розуміння структури проєкту  
- опису загальних правил  
- навігації по інфраструктурі  
- пояснення зв’язків між сервісами  

---

## 🧭 Архітектура інфраструктури

Репозиторій поділений на логічні домени:

- 🌐 Network → мережеве обладнання  
- 🔐 VPN → віддалений доступ  
- 📊 Monitoring → системи моніторингу  
- 🐧 Linux → серверні системи  
- 🪟 Windows → Windows Server  
- 🔑 Auth → аутентифікація  
- 👥 Directory → Active Directory  
- ☁️ Cloud → Microsoft 365  
- 🔗 Integrations → зв’язки між сервісами  

---

## 📏 Загальні стандарти

- кожен сервіс має однакову структуру (docs/, configs/, scripts/, backups/)  
- документація пишеться у форматі Markdown  
- назви файлів тільки lowercase з -  
- заборонено зберігати секрети у відкритому вигляді  

---

## ⚙️ Принципи роботи

- Infrastructure as Code  
- документація передує реалізації  
- мінімум ручних дій  
- автоматизація всього, що можливо  

---

## 🔐 Безпека

- паролі та ключі не зберігаються в репозиторії  
- використовується принцип least privilege  
- доступ до сервісів контролюється через централізовану аутентифікацію  
- всі зміни повинні бути перевірені перед застосуванням  

---

## 🔗 Інтеграції

Сервіси не працюють ізольовано. Основні зв’язки:

- OpenVPN ↔️ RADIUS ↔️ Active Directory  
- Monitoring ↔️ всі сервіси  
- Cloud ↔️ локальна інфраструктура  

---

## 📂 Як користуватись репозиторієм

1. Знайди потрібний домен (network, vpn, linux…)  
2. Перейди у відповідну папку  
3. Відкрий README.md  
4. Далі переходь у docs/ для деталей  

---
